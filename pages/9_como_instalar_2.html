¿Cómo se instala el sistema de modelado de tópicos? Parte 2 - Freeling

  • Lo primero es descargar el freeling, en su página oficial http://nlp.lsi.upc.edu/freeling/node/30, por motivos de este taller esta disponible en dirección http://192.168.12.65/freeling/
  • Según la documentación oficial del freeling ubicada en https://talp-upc.gitbooks.io/freeling-user-manual/content/index.html, las dependencias son las siguientes:
    • sudo apt-get install libboost-regex-dev libicu-dev zlib1g-dev libboost-dev
    • sudo apt-get install libboost-system-dev libboost-program-options-dev libboost-thread-dev
      • Con estas dependencias instaladas, descomprimimos el archivo del freeling, entramos con la consola y ejecutamos:
        • autoreconf --install
        • ./configure
        • make
        • sudo make install
      • Si al hacer esto no obtuvimos errores, ya estamos listos, ahora si nos da error con el ./configure o make será un dolor de cabeza.
      • Como consuelo, instalamos el freeling 4.0 en una instalación limpia de debian 9 y no tuvimos problemas.
      • Con esto listo, el freeling se debió haber instalado en una ruta similar a esta /usr/local/share/freeling/, al menos en debian lo debió hacer
      • Ya sólo nos queda añadir la ruta a las variables de entorno, en el ~/.bashrc agregamos al final la linea export FREELINGSHARE="/usr/local/share/freeling/" recordemos que la ruta puede variar.
      • Si ejecutamos esto en consola /usr/local/bin/analyzer -f /usr/local/share/freeling/config/es.cfg y nos da este error: LC_ALL: cannot change locale (en_US.UTF-8): No such file or directory. lo podemos solucionar ejecutando el comando sudo dpkg-reconfigure locales, y agregar el paquete en_US.UTF-8
      • Ya con eso estamos listos con este largo y engorroso paso.
